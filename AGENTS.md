# AGENTS.md

Проект: сравнение наивного алгоритма betweenness centrality и Brandes для невзвешенного неориентированного связного графа на Chapel.

Требования:
- Язык: Chapel.
- Совместимость: сначала Chapel 2.8, по возможности не использовать фичи новее Chapel 2.0.
- Граф: невзвешенный, неориентированный, одна компонента связности.
- Генерация: случайный граф, с параметрами командной строки `--n` и `--seed`.
- Импорт графов из файлов пока не нужен.
- Вывод отчёта: только в stdout.
- Нужны две команды уровня пользователя:
  - Generate
  - Run
- Нужен Makefile.
- Нужны unit-тесты на малых графах.
- Нужен полностью воспроизводимый pipeline.
- Корректность важнее производительности.
- Нельзя дублировать большие структуры графа между потоками/задачами без крайней необходимости.
- Представление графа: сначала CSR; если ELLPACK даст явное преимущество и не усложнит код, можно обсудить, но по умолчанию использовать CSR.
- Сравнение:
  - baseline: наивный алгоритм;
  - optimized: Brandes;
  - затем сравнение результатов на точное совпадение.
- Для профилирования фиксировать:
  - размер графа,
  - seed,
  - время генерации,
  - время каждого алгоритма,
  - общее время,
  - PASS/FAIL сравнения.
- На первом этапе сделать последовательные версии.
- На втором этапе добавить параллельный Brandes через coforall и деление источников на блоки.
- Многоузловой запуск пока не реализовывать агрессивно; архитектуру не ломать, чтобы потом можно было расширить.
